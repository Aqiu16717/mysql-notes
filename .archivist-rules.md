# 投稿规范 — mysql-notes 贡献者指南

> **适用对象：** 所有向 mysql-notes 投稿的人（人类和 Agent）。  
> **执行者：** @mns-archivist。  
> **最后更新：** 2026-05-16。

---

## 文件命名

```
YYYY-MM-DD-{product}-{topic}.md
```

示例：
- `2026-05-16-aurora-storage-layer-architecture.md`
- `2026-05-16-percona-mvcc-read-view.md`
- `2026-05-16-cross-product-replication-comparison.md`

**文件名中不使用空格，用连字符。**

---

## Front Matter（必填）

每篇笔记**必须**在文件最顶部包含以下 YAML Front Matter：

```yaml
---
title: "笔记标题"
date: YYYY-MM-DD
tags: [产品, 模块, 类型, 难度]
source: [url1, url2]
version: "MySQL 9.6.0 / Aurora 3.04 / Percona 8.0.33"
author: "@你的handle"
---
```

### 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `title` | ✅ | 人类可读的标题，使用句子大小写 |
| `date` | ✅ | 笔记创建日期（不是来源发布日期） |
| `tags` | ✅ | 数组：`[#产品, #模块, #类型, #难度]`。详见 `notes/tags.md` |
| `source` | ✅ | 至少一个 URL 或引用。多个来源用数组。必须标注来源类型（见下方示例） |
| `version` | ✅ | 笔记所依据的具体版本号。不要写 "MySQL 8.0"，要写 "MySQL 8.0.37" |
| `author` | ✅ | 你的 @handle |

### source 字段格式

```yaml
source:
  - type: source-read        # 源码分析来源
    ref: "@mns-reader: THD 状态机分析 (2026-05-16)"
  - type: paper-card         # 论文卡片来源
    ref: "@mns-hunter: 2026-05-16-sigmod-hekaton.md"
  - type: bulletin           # 情报简报来源
    ref: "@mns-scout: 2026-05-16-ecosystem-scan.md"
  - url: "https://..."       # 外部 URL（无 type 时为通用引用）
```

必须标注 `type` 以便归档器自动生成素材追溯链。

### 标签速查

详见 `notes/tags.md`。快速参考：

- **产品：** `#mysql` `#aurora` `#percona` `#mariadb` `#heatwave` `#tidb` `#oceanbase` `#frankenmysql`
- **模块：** `#innodb` `#binlog` `#replication` `#optimizer` `#executor` `#parser` `#dd` `#ddl` `#server`
- **类型：** `#source-code` `#architecture` `#paper` `#comparison` `#tutorial` `#learning-card` `#experiment` `#faq` `#bulletin`
- **难度：** `#beginner` `#intermediate` `#advanced`

## 笔记放哪里

**KISS 原则：所有笔记直接放 `notes/` 目录，不建子目录。**

```
notes/YYYY-MM-DD-{product}-{topic}.md
```

标签（产品、模块、类型、难度）在 Front Matter 中承担全部分类职责。`INDEX.md` 是全局导航入口。

---

## 交叉引用

在笔记正文中使用以下约定：

```
参见：[[notes/2026-01-15-aurora-buffer-pool.md]]
对比：[[notes/2026-02-20-percona-buffer-pool.md]]
来源：@mns-hunter 论文卡片：2026-05-16-sigmod-hekaton.md
```

归档器在 MOC 索引页中维护双向链接。

---

## 禁止事项

1. ❌ **禁止分目录存放**——所有笔记直接放 `notes/` 下，不要建子目录
2. ❌ **禁止标签缺失**——每篇笔记必须包含产品、模块、类型、难度四类标签
3. ❌ **禁止 source 为空**——必须标注信息来源。无源信息 = 观点，不是笔记
4. ❌ **禁止模糊版本号**——写 "MySQL 9.6.0" 而不是 "MySQL 9.x" 或 "MySQL 8.0"
5. ❌ **禁止同义多标签**——不要同时用 `#innodb` 和 `#inno-db`。查 `notes/tags.md`
6. ❌ **禁止纯文字长文**——复杂概念需要图表或代码段。纯文字超过三屏必须拆分

---

## 提交流程

1. 写好笔记，补全 Front Matter
2. 放到 `notes/` 目录下
3. 在 #mysql-notes 频道 @mns-archivist 请求审核
4. 归档器验证 Front Matter、标签、存放位置
5. 归档器更新 INDEX.md 和 MOC 索引
6. 归档器在下次《索引更新摘要》中报告
