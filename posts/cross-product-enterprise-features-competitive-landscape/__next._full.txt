<h2 id="3-竞争影响分析谁受损最大谁基本不受影响">3. 竞争影响分析：谁受损最大，谁基本不受影响</h2>
<h3 id="-percona-server--受损最大">🔴 Percona Server — 受损最大</h3>
<p><strong>Percona 的历史定位是："MySQL 社区版 + 企业功能，免费提供"。</strong> 8 项功能下放直接侵蚀了这个定位：</p>
<table>
<thead>
<tr>
<th>Percona 差异化优势</th>
<th>MySQL 9.7 后的状态</th>
</tr>
</thead>

<tr>
<td>扩展性能计数器 (853 vs 434)</td>
<td>仍领先，但差距缩小</td>
</tr>
<tr>
<td>MyRocks 存储引擎</td>
<td>仍为 Percona 独有</td>
</tr>
<tr>
<td>线程池</td>
<td>MySQL CE 仍缺失——关键差异化点</td>
</tr>
<tr>
<td>PAM/LDAP 认证</td>
<td>MySQL CE 仍缺失——关键差异化点</td>
</tr>
<tr>
<td>审计插件</td>
<td>MySQL CE 仍缺失——关键差异化点</td>
</tr>
<tr>
<td>TDE 透明加密</td>
<td>MySQL CE 仍缺失——关键差异化点</td>
</tr>
<tr>
<td>Hypergraph 优化器</td>
<td>MySQL CE 现在有且默认开启——<strong>优势消失</strong></td>
</tr>

<td>复制监控指标</td>
<td>MySQL CE 现在有——<strong>优势消失</strong></td>

<p><strong>结论：</strong> Percona 的价值主张从约 12 项独特特性缩减到约 5 项。线程池 + TDE + 审计 + PAM + MyRocks 是最后的护城河。</p>

<h3 id="-tidb--基本不受影响">🟢 TiDB — 基本不受影响</h3>
<p>TiDB 在完全不同的维度上竞争——分布式 SQL + 水平扩展。MySQL 9.7 CE 完全不触及这个领域：</p>

<td>MySQL 需 HeatWave（仅云端、额外付费）</td>

<p><strong>MySQL 9.7 对 TiDB 唯一获得优势的点：</strong> TiDB 无原生 JSON Duality 视图，优化器非超图引擎。MySQL 在单节点查询优化上优于 TiDB。</p>
<h3 id="-oceanbase--基本不受影响">🟢 OceanBase — 基本不受影响</h3>
<p>OceanBase 社区版免费开源且具备 MySQL CE 仍无法匹敌的能力：</p>

<h2 id="6-总体判断">6. 总体判断</h2>
<p><strong>MySQL 9.7 CE 是 Oracle 给社区版的最好单节点 MySQL。</strong> Hypergraph 优化器本身就是对旧优化器的代际跨越。JSON Duality 视图确实是独门武器。</p>
<p>但两个结构性问题仍然存在：</p>
<ol>
<li><strong>VECTOR 类型是"语法糖"——有类型无 HNSW 索引</strong>。能存向量但不能做快速相似度搜索。OceanBase 社区版免费提供 HNSW。</li>
<li><strong>HTAP 仍然锁定在 HeatWave（付费）</strong>——而 MariaDB 和 OceanBase 都有免费的列存引擎。</li>
</ol>
<p><strong>给 mysql-notes 读者的建议：</strong></p>
<ul>
<li>MySQL 原生用户需要优化器提升 → 立刻升级到 9.7 CE</li>
<li>需要 TDE/审计但不向 Oracle 付费 → Percona 8.4</li>
<li>需要 HTAP 或向量搜索且不想云锁定 → OceanBase 4.3 CE 或 MariaDB 11.8</li>
<li>需要水平扩展能力 → TiDB 或 OceanBase（均为开源免费）</li>
